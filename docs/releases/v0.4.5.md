## v0.4.5 — pre-submission supply chain polish

Prep patch for Anthropic plugin marketplace resubmission. Supply chain posture is now surfaced as prominently as the core functionality. No behavior change.

### Added

- **`SECURITY.md`**: vulnerability reporting policy with a stated 72-hour SLA on acknowledgment, private channel via `cj@stetkeep.com`, and declared in-scope vs out-of-scope reports. Documents the supply chain stack explicitly:
  - OIDC Trusted Publisher (no long-lived `NPM_TOKEN`)
  - Sigstore SLSA v1 provenance on every tarball
  - SHA-pinned GitHub Actions
  - Zero runtime dependencies (stdlib-only)
- **Sigstore provenance badge** on the README, version-agnostic link to the npm package page.
- **"Security & supply chain" section** on the README between Contributing and License, with the five concrete guarantees.
- **GitHub issue templates** (`bug.yml`, `false_positive.yml`, `config.yml`) — the false-positive template directly targets the catalog's highest-leverage contribution type.
- `SECURITY.md` ships in the npm tarball (added to `package.json` `files` array) so users reading the npm page also see the policy.

### Changed

- `lib/install.js`: `claude-brain` removed from the bootstrap detection regex. `claude-brain` was a pre-launch prototype name with no installs; `mdbrain` and `stetkeep` remain for legitimate migration detection.

### No functional changes

CLI behavior, hook decisions, plugin manifest, subagent tools, protocol structure, false-positive catalog entries: identical to v0.4.4.

### Migration

No user action required. `npm install -g stetkeep` auto-upgrades 0.4.4 → 0.4.5.
