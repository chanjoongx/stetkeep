## 🚀 What's new

mdbrain is now distributed via two official channels:

### npm (live)
```bash
npx mdbrain install
```
Zero-clone, one-command install. Works on macOS/Linux/Windows with Node 20+.
Published at https://www.npmjs.com/package/mdbrain (zero runtime dependencies).

### Claude Code plugin marketplace (self-hosted)
```
/plugin marketplace add chanjoongx/mdbrain
/plugin install mdbrain@mdbrain-marketplace
```
Native 2026 Claude Code extension — passes `claude plugin validate` on both
`marketplace.json` and `plugin.json`.

## ✨ Key additions

- **`bin/mdbrain.js`** CLI entry (ESM, Node 20+, stdlib-only)
- **`lib/install.js`** — cross-platform installer with coexist / merge / fresh modes
- **`lib/scan.js`** — read-only MD ecosystem diagnostic
- **`.claude-plugin/plugin.json`** — plugin manifest (validator-approved)
- **`.claude-plugin/marketplace.json`** — self-hosted marketplace catalog
- **`hooks/hooks.json`** — PreToolUse hook manifest (auto-loaded)
- **Plugin-convention layout** — `agents/`, `commands/`, `hooks/` at plugin root

## 🔄 Positioning reset

Competitive research revealed "mechanical guardrails via hooks" and "tool-scoped
subagents" are table-stakes in April 2026 (TDD-Guard, VoltAgent etc. already
own those). Repositioned around two genuinely unique pillars:

- **XML-structured protocol framework** — BRAIN/CRAFT/PERF treat Claude's
  attention surface as addressable via XML tags (per Anthropic's own prompting
  recommendations)
- **False-positive catalog** — 16-entry registry of "patterns that look like
  anti-patterns but aren't" (1000-line configs, V8 JIT paths, framework
  idioms, etc.)

No competitor ships either of these as a first-class feature.

## 🛠 Undocumented plugin quirks discovered

Via real `claude plugin install` testing (not in official docs):

1. **`agents` field must be a file array, not a directory string.**
   `"agents": "./agents/"` is rejected by validator. Use
   `"agents": ["./agents/brain-router.md", ...]` instead.
2. **`hooks/hooks.json` auto-loads** — do NOT reference it from `plugin.json`.
   Listing `"hooks": "./hooks/hooks.json"` causes a
   "Duplicate hooks file detected" error at plugin load time.
3. **`$schema` URLs are strictly validated** — settings.json must use
   `https://json.schemastore.org/claude-code-settings.json`, not other URLs.
   Mismatch silently skips the entire settings file.

All three are fixed in this release.

## 🔧 Fixed Windows pain points

- PowerShell execution policy: `npx mdbrain` bypasses it entirely
- em-dash rendering corruption on Korean CP949 consoles
- `bash` in PowerShell → WSL proxy collision avoided by using Node CLI

## 📦 Install

```bash
# Option 1 — npm (recommended)
npx mdbrain install

# Option 2 — Claude Code plugin marketplace
/plugin marketplace add chanjoongx/mdbrain
/plugin install mdbrain@mdbrain-marketplace

# Option 3 — git clone (for contributors)
git clone https://github.com/chanjoongx/mdbrain.git
bash /path/to/mdbrain/install.sh
```

After install, in Claude Code:
```
/brain-scan
```
