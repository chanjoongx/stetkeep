## v0.4.4 — OIDC workflow fix (Node 24 + npm v11+)

Patch release that fixes the GitHub Actions publish workflow. No package content changes vs v0.4.3.

### Root cause of v0.4.2 / v0.4.3 auto-publish failures

Both releases failed with a misleading `404 Not Found - stetkeep is not in this registry` error *after* Sigstore provenance signing succeeded. Deep research uncovered the actual cause:

- `actions/setup-node@v4` with `node-version: 22` ships **npm v10.x**
- npm v10.x does **not support** the OIDC handshake required by npm's Trusted Publisher flow
- npm CLI must be **>= 11.5.1** for Trusted Publishing to work
- The 404 is how npm signals a failed OIDC handshake, not an actual missing-package error

Source: [npm/cli#8976](https://github.com/npm/cli/issues/8976)

### Fix

`.github/workflows/publish.yml`:

- `node-version: '22'` → `'24'` (Node 24 ships with npm v11)
- Added explicit `npm install -g npm@latest` step (belt-and-suspenders)
- Header comments updated to document the requirement

### If this release publishes successfully

- Trusted Publisher config + Node 24 confirmed as the working combination
- Provenance attestation auto-generated and linked on npm page
- All future releases: `git tag vX.Y.Z && git push origin vX.Y.Z` one-liner

### Migration

No user action. `npm install -g stetkeep` auto-upgrades.
