## 🔧 v0.4.3 — protocol accuracy fixes + first OIDC end-to-end validation

Three Should-fix items from the 6-expert autonomous audit, plus a deliberate test of the OIDC trusted-publisher pipeline.

### 🎯 Protocol content corrections

- **`CRAFT.md` A20** "Comment instead of refactor": example changed from `// TODO: careful` to `// HACK: works but fragile`. Plain `// TODO` referencing an upstream fix is legitimate and not A20. The previous example conflated TODOs with actual code-smell markers.
- **`PERF.md` P9** "No input debounce": transform refined. `useMemo(() => debounce(fn, 300), [])` alone leaks timers on unmount. Correct pattern is `useMemo` + `useEffect` cleanup, or adopt `use-debounce`.
- **`PERF.md` P20** "Unnecessary SSR re-hydration": transform updated. "Selective hydration" is React 18's automatic behavior, not a user-applied transform. Correct user-controllable alternatives are islands architecture (Astro / Qwik) or trimming `use client` boundaries.

### 🧪 OIDC trusted-publisher end-to-end test

This is the **first release to ship exclusively via the `.github/workflows/publish.yml` OIDC pipeline** (no manual `npm publish` on the local machine). v0.4.2 attempted the same but a parallel manual publish won the race, leaving the Actions workflow to 404 on PUT.

**If this release succeeds**: Trusted Publisher config confirmed correct, provenance attestation auto-linked to the npm page, all future releases become `git tag + git push` one-liners.

**If it fails** with the same 404: v0.4.2's failure wasn't a race condition; it's a genuine config issue and we debug further.
