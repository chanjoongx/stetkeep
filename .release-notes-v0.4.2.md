- **README Roadmap**: v0.4 marked complete (rename + OIDC + Sigstore provenance). v0.5 and v0.6 re-mapped (benchmark + `npx stetkeep init` for v0.5, per-language variants for v0.6).

- **README FAQ "Why stetkeep?"**: shortened. Removed the defensive paragraph about the IBM 1977 STET text editor and STET SA European payment processor (too much "we are not those" for a project that doesn't actually need the disclaimer).
- **README competitive-research link** retargeted to a more useful anchor.

### 🔄 Migration

No user action. `npm install -g stetkeep` auto-upgrades 0.4.1 → 0.4.2.

No functional changes vs v0.4.1. CLI behavior, hook semantics, subagent routing, protocol content all unchanged.

### False alarms intentionally skipped (from the 6-agent audit)

To prevent confusion for contributors reading the CHANGELOG, a few audit findings were verified as **false alarms** and intentionally NOT applied:

- plugin.json `commands`/`agents` as "invalid fields" — actually valid per the official marketplace schema
- plugin.json `version` "missing" — already present
- `disallowedTools` / `permissionMode` / `paths:` frontmatter keys as "spec violations" — all three are documented subagent/rules frontmatter fields per Anthropic's 2026 Claude Code docs
- plugin.json `repository` bare string as blocker — both string and object forms are valid per npm/plugin spec
- install.js legacy-bootstrap regex as blocker — the broad match is intentional (catches `mdbrain` / `claude-brain` legacy headers, preventing duplicate bootstrap appends)

Fresh fact-checking against official docs before applying fixes saved ~9 unnecessary edits.
