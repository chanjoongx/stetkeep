## 🔄 Renamed: mdbrain → stetkeep

v0.4.0 is a **rename release**. The project formerly known as `mdbrain` is now **`stetkeep`**. Functionality is identical to v0.3.0; only the name and URLs change.

### Why the rename

[mediaire's `mdbrain`](https://mediaire.ai/en/mdbrain/) is a CE-certified medical AI product for brain MRI analysis (certified 2019-01-21), with six years of clinical commercial deployment across Europe. Our `mdbrain` and their `mdbrain` target different users and markets, but sharing the name creates real risk:

- Trademark dilution / confusion claims from an established medical brand
- Anthropic marketplace reviewer rejection on name-collision discovery
- SEO collision: "mdbrain" searches are dominated by the medical product

The rebrand was executed one day after the v0.3.0 launch, when the npm download count was effectively zero — the cheapest possible moment to switch.

### The new name

**`stet`** is the traditional editorial mark from Latin *"let it stand"*. Editors write `stet` over text that's been marked for deletion to cancel the deletion and preserve the original. **`stetkeep`** applies the same principle to code: stop Claude from "helpfully" refactoring your intentional code.

```bash
npm install stetkeep
```

The `stetkeep` package registers `stet` as a bin alias, so after install both `stet install` and `stetkeep install` work. (We attempted to publish an additional `stet` npm package as a shorthand but npm's typosquatting policy blocks the name, which also means no one else can claim it; the CLI shorthand is what matters in practice.)

### Migration for `mdbrain@0.3.0` users

Your user base is effectively zero (launch was yesterday), but if you managed to install `mdbrain` in the last 24 hours:

```bash
# Uninstall old
npm uninstall -g mdbrain

# Install new
npm install -g stetkeep
```

The old `mdbrain@0.3.0` package has been deprecated on npm with a pointer to this new package.
